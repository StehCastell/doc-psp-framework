# Design Review Checklist

## Objective

Assess the quality of the design solutions before implementation.

---

## Architecture

- [ ] Solution compliant with the defined architecture.
- [ ] Well-defined interfaces.
- [ ] Justified dependencies.

---

## Requirements

- [ ] Requirements covered.
- [ ] Exceptional cases considered.
- [ ] Acceptance criteria analyzed.

---

## Quality

- [ ] Adequate complexity.
- [ ] Reuse possibility assessed.
- [ ] Impacts on other modules considered.

---

## Security

- [ ] Risks identified.
- [ ] Sensitive data considered.
- [ ] Necessary controls planned.

---

## Review Result

- [ ] Approved
- [ ] Approved with remarks
- [ ] Rejected
